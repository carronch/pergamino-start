# pergamino-heartbeat

Cloudflare Worker for the pergamino client-deployment heartbeat.

**One Worker per client.** Provisioned by `pergamino-deploy new <slug>` from this template.

## What it does

Three crons + three webhooks. Together they make pergamino "alive."

| Cron | When (local time) | What |
|---|---|---|
| `reviewer-digest` | Weekday early-morning local | Pre-analyzes pending PRs in `brain-canon`. Composes WhatsApp digest. Sends to the principal + the operator + dept heads per CODEOWNERS. |
| `user-nudge` | Weekday evening local | Scans each user's workspace for un-promoted items. Sends per-user WhatsApp nudge. |
| `customer-bot-weekly` | Sunday evening local | Reviews customer-facing knowledge gaps from the past week. Proposes FAQ updates as PRs (principal + operator review). |

| Webhook | What |
|---|---|
| `POST /whatsapp-reply` | Handles BULK APPROVE / N APPROVE / N REJECT / SNOOZE replies from reviewer digests. |
| `POST /whatsapp-customer` | Inbound concierge messages from real customers. Loads `customer-facing/` context, replies in matching language, escalates the 20%. |
| `POST /github` | PR open/sync events. Optionally runs reviewer-AI immediately so verdicts appear in the PR before the next digest cron. |

## Compute economics

The Worker calls Anthropic's API with the **client's own API key** (set via `wrangler secret put ANTHROPIC_API_KEY`). Per-PR cost:

- **Sonnet** (default for `customer-facing/` and `finance/` scopes): ~$0.05 per PR
- **Haiku** (default for routine within-scope): ~$0.005 per PR

For a 12-person client running ~50 PRs/week + ~200 concierge conversations/week: **~$15–30/month** of API spend. Worker hosting itself is free (well within Cloudflare's free tier).

## Why per-client Worker (vs. one multi-tenant Worker)

- **Blast-radius isolation**: deleting a client = deleting one Worker. Clean.
- **Cost transparency**: each client sees their own bill.
- **Secret isolation**: each client's GitHub token / Anthropic key is in their own Worker's secrets.
- **Deploy independence**: upgrading the heartbeat for one client doesn't affect others.

The trade-off: the operator maintains N copies of the same code. Solution: this template is the **source of truth**; per-client deploys are auto-generated by `pergamino-deploy`. Updates propagate via `pergamino-deploy upgrade <client>`.

## Setup (manual, until `pergamino-deploy` is built)

```sh
# 1. Provision Cloudflare resources
wrangler kv:namespace create pergamino-<client-slug>
wrangler d1 create pergamino-<client-slug>

# 2. Update wrangler.toml with the IDs from the above commands.

# 3. Set secrets
wrangler secret put ANTHROPIC_API_KEY        # client's Anthropic console key
wrangler secret put GITHUB_TOKEN             # fine-grained PAT, brain-canon read+write
wrangler secret put WHATSAPP_TOKEN           # Meta Cloud API token
wrangler secret put WHATSAPP_PHONE_ID
wrangler secret put WHATSAPP_VERIFY_TOKEN
wrangler secret put REVIEWER_AGENTS_REPO     # {{ORG}}/pergamino-reviewer-agents@main

# 4. Initialize the D1 schema
wrangler d1 execute pergamino-<client-slug> --file ./schema.sql

# 5. Configure WhatsApp webhooks in Meta Business Manager
#    URL: https://pergamino-heartbeat-<slug>.<your-cf-account>.workers.dev/whatsapp-reply
#    Verify token: same as WHATSAPP_VERIFY_TOKEN secret

# 6. Deploy
wrangler deploy
```

## D1 schema

Initial tables (`schema.sql` should contain):

```sql
CREATE TABLE IF NOT EXISTS pr_reviews (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  pr_number INTEGER NOT NULL,
  verdict TEXT NOT NULL,
  confidence TEXT NOT NULL,
  reasoning TEXT,
  flags TEXT,  -- JSON array
  ts INTEGER NOT NULL
);

CREATE TABLE IF NOT EXISTS escalations (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  phone TEXT NOT NULL,
  last_message TEXT NOT NULL,
  ts INTEGER NOT NULL,
  resolved INTEGER DEFAULT 0
);

CREATE TABLE IF NOT EXISTS knowledge_gaps (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  question TEXT NOT NULL,
  ts INTEGER NOT NULL,
  proposed_pr INTEGER  -- nullable; set if a FAQ proposal PR was opened
);

CREATE INDEX IF NOT EXISTS idx_kg_ts ON knowledge_gaps(ts);
CREATE INDEX IF NOT EXISTS idx_pr_ts ON pr_reviews(ts);
```

These stubs are explicit (search for `// Simplified placeholder`).

## Architecture references

- The reviewer-agent prompts the Worker calls live in a separate repo (template: `{{ORG}}/pergamino-reviewer-agents`). One file per scope.
- The brain-canon schema (the folders the Worker reads PRs against) is generated by `pergamino-intake` skill + `pergamino-deploy` from a per-client YAML scope plan.
- Worker → Anthropic uses Sonnet for high-stakes scopes, Haiku for routine. Configurable per-client in `wrangler.toml`.

## Cost discipline

Watch the per-client AI spend dashboard. Two early-warning signals:

1. **Sonnet calls > 200/day** — likely the cheap-tier classifier is bleeding. Re-tune the model-per-scope rule.
2. **Reviewer-AI verdict accuracy < 80%** (vs human override) — first-3-weeks calibration not converging; pause and retune the per-scope reviewer prompts.

Both are visible in the pergamino dashboard tile for each client.
